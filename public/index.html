Cómo funciona

¿Cómo está hecha esta tienda?

Una aplicación web completa construida desde cero con Node.js y PostgreSQL. Sin frameworks complejos. Cada pieza está aquí por una razón concreta.

El servidor web

El backend corre en Node.js con Express. Expone una API REST que el frontend consume: productos, registro, login y tasas de cambio. Todo vive en server.js.

Dónde viven los datos

Usuarios y productos viven en PostgreSQL. Las contraseñas nunca se guardan en texto plano: se hashean con bcrypt antes de almacenarse.

Cómo funciona el login

Al iniciar sesión el servidor devuelve un token JWT de 7 días. El navegador lo guarda en localStorage y lo adjunta a cada petición protegida.

Precios en tiempo real

El servidor consulta frankfurter.app y convierte precios a EUR y MXN. El navegador nunca llama a esa API directamente; todo pasa por /api/rates.

El carrito de compras

El carrito persiste entre recargas porque se guarda en localStorage. No necesita base de datos: es estado del navegador.

El frontend

Sin React ni Vue. HTML estático, Tailwind vía CDN y todo el comportamiento dinámico en un solo archivo app.js.
